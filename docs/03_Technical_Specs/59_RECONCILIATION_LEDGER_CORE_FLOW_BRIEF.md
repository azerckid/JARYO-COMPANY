# Reconciliation Ledger Core Flow Brief

> Created: 2026-07-14  
> Status: Slice B implementation contract

## 0. 목적

자료대조원장은 업로드한 거래를 기간별로 찾고, 중복·누락·증빙·계정 문제를 해결해 신고에 사용할 확정 원장을 만드는 작업 화면이다. 설명용 대시보드가 아니라 반복 작업에 바로 쓰는 표 중심 화면이어야 한다.

## 1. 화면 계약

첫 화면에는 다음만 둔다.

1. 현재 기간과 이전·다음 기간 이동
2. 월·분기·반기 단위 전환
3. 출처·처리 상태 탭
4. 거래처·금액·적요 검색
5. 안전한 계정 패턴 일괄 확정
6. 원장 표와 중복 의심 해결 액션
7. 마감 체크리스트와 세목별 차단 이유

Hero, 출처 요약 카드, 다음 할 일 카드, 비활성 표시 설정, 비활성 선택 확정, 사용자 지정 날짜 입력은 두지 않는다. 지원되지 않는 조작을 보여주지 않는다.

## 2. 기간 계약

- 서버가 이미 지원하는 `YYYY-MM`, `YYYY-Q1..Q4`, `YYYY-H1..H2`만 노출한다.
- 월·분기·반기 전환은 현재 기간의 가운데 월을 기준으로 같은 시점의 새 기간을 선택한다.
- 이전·다음은 현재 단위에서 한 칸 이동한다.
- 기간을 바꿔도 현재 출처 탭과 fixture 여부는 유지한다.
- 연간·사용자 지정은 조회 계약이 생기기 전까지 표시하지 않는다.

## 3. 검색 계약

- 검색은 현재 서버 기간과 현재 탭 안에서 즉시 적용한다.
- 거래처, 적요, 날짜, 금액, 확정·추천 계정항목을 검색한다.
- 쉼표와 공백을 정규화해 `628,316`과 `628316`을 같은 금액으로 찾는다.
- 검색어가 없으면 기존 정렬을 그대로 유지한다.

## 4. 중복 의심 계약

### 4.1 후보 진입

다음 값이 모두 같은 미제외 거래 2건 이상만 중복 의심으로 표시한다.

- 출처
- 입출금 방향
- 거래일
- 절대 금액
- 정규화한 거래처
- 정규화한 적요

거래처나 적요가 비어 있으면 중복 의심으로 만들지 않는다. 이 규칙은 자동 제외가 아니라 사용자 확인 대상으로만 사용한다.

### 4.2 해결

`중복 확인`을 누르면 현재 거래와 같은 그룹의 거래를 한 모달에서 비교한다.

- `별도 거래로 확인`: 기존 메모를 보존하고 `중복 검토: 별도 거래로 확인` 감사 문구를 추가한다.
- `현재 거래를 중복 제외`: `제외 사유: 중복 증빙 - 동일 날짜·금액·거래처·적요`로 제외한다.
- 두 mutation 모두 기존 최근 작업 되돌리기 계약을 재사용한다.

한 행을 별도 거래로 확인하거나 제외하면 남은 미확정 행만 다시 그룹화한다. 2건 그룹은 즉시 해소되고, 3건 이상 그룹은 중복 가능성이 남은 행만 계속 표시한다.

## 5. 신고 게이트

- 미해결 중복 의심 행 수를 `duplicateReviewCount`로 계산한다.
- 이 수는 자료대조원장 `blockerCount`와 `isReadyForPath1`에 포함한다.
- 신고 준비 화면도 동일한 shared reconciliation gate 값을 읽는다.
- 중복 의심은 계정·증빙 판단을 자동으로 덮어쓰지 않는다.

## 6. Side-Effect Isolation

- 기존 세액 계산, 증빙 연결, 계정 확정, 제외·소명 mutation 계약은 변경하지 않는다.
- 신규 DB 컬럼이나 migration 없이 기존 `staffMemo` 감사 접두사 계약을 재사용한다.
- 분류 큐의 excluded 숨김 규칙은 유지하고, 자료대조원장만 제외 행을 계속 표시한다.
- fixture는 시각 검증용이며 저장은 계속 비활성이다.

## 7. 완료선

- 기간 단위 전환과 이전·다음 이동이 실제 서버 조회를 바꾼다.
- 탭 이동 시 기간이 유지되고 검색이 즉시 작동한다.
- 중복 의심 탭의 건수와 실제 행 수가 일치한다.
- 별도 거래 확인과 중복 제외 후 해당 그룹과 신고 게이트가 재계산된다.
- 미해결 중복이 있으면 Path 1 준비 완료가 되지 않는다.
- TypeScript, 전체 테스트, lint, whitespace, Chrome 데스크톱·모바일 검증을 통과한다.
